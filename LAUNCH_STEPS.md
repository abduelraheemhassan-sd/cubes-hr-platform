# خطوات إطلاق منصة CubesHR - دليل شامل

## 📌 نظرة عامة

هذا الدليل يوضح الخطوات العملية لإطلاق منصة CubesHR بنجاح، مقسمة إلى مراحل واضحة مع تفاصيل كل خطوة.

## المرحلة الأولى: التحضير والإعداد (الأسابيع 1-2)

### الخطوة 1: إعداد البنية التحتية

#### 1.1 اختيار مزود الاستضافة
اختر مزود استضافة موثوق يوفر الخدمات التالية:
- خوادم قوية (2GB RAM على الأقل)
- قاعدة بيانات MySQL مدارة
- نسخ احتياطية تلقائية
- دعم فني 24/7

**الخيارات الموصى بها**: AWS, DigitalOcean, Linode, Heroku

#### 1.2 إعداد الخادم
```bash
# تثبيت Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# تثبيت npm/pnpm
npm install -g pnpm

# تثبيت قاعدة البيانات
sudo apt-get install mysql-server

# تثبيت Redis
sudo apt-get install redis-server
```

#### 1.3 إعداد متغيرات البيئة
أنشئ ملف `.env.production` بالمتغيرات التالية:
```
NODE_ENV=production
DATABASE_URL=mysql://user:password@host/cubes_hr
REDIS_URL=redis://localhost:6379
JWT_SECRET=your-secret-key
OAUTH_SERVER_URL=https://oauth.example.com
VITE_APP_ID=your-app-id
VITE_OAUTH_PORTAL_URL=https://login.example.com
```

### الخطوة 2: إعداد قاعدة البيانات

#### 2.1 إنشاء قاعدة البيانات
```sql
CREATE DATABASE cubes_hr;
CREATE USER 'cubes_user'@'localhost' IDENTIFIED BY 'strong_password';
GRANT ALL PRIVILEGES ON cubes_hr.* TO 'cubes_user'@'localhost';
FLUSH PRIVILEGES;
```

#### 2.2 تشغيل الهجرات
```bash
cd /home/ubuntu/cubes-hr-platform
pnpm db:push
```

#### 2.3 إنشاء حساب المسؤول الأول
```bash
# استخدم واجهة الإدارة أو قم بتشغيل سكريبت الإنشاء
pnpm ts-node scripts/create-admin.ts
```

### الخطوة 3: إعداد الأمان

#### 3.1 تثبيت شهادة SSL
```bash
# استخدام Let's Encrypt
sudo apt-get install certbot python3-certbot-nginx
sudo certbot certonly --standalone -d yourdomain.com
```

#### 3.2 إعداد جدار الحماية
```bash
sudo ufw enable
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

#### 3.3 إعداد HTTPS Redirect
تأكد من إعادة توجيه جميع طلبات HTTP إلى HTTPS.

### الخطوة 4: الاختبارات الأولية

#### 4.1 تشغيل الاختبارات
```bash
pnpm test
```

#### 4.2 اختبار الأداء
```bash
# استخدام أداة مثل Apache Bench
ab -n 1000 -c 100 https://yourdomain.com
```

#### 4.3 اختبار الأمان
```bash

npm audit
```

### الخطوة 5: الإطلاق على بيئة التجريب

#### 5.1 نشر التطبيق
```bash

pnpm build

# نسخ الملفات إلى الخادم

```

#### 5.2 إعداد قاعدة البيانات التجريبية
