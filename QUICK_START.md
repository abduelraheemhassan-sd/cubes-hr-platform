# البدء السريع - CubesHR

## ⚡ تشغيل سريع (5 دقائق)

### الطريقة 1: التشغيل المحلي (الأسرع)

```bash
# 1. استنساخ المشروع
git clone https://github.com/your-org/cubes-hr-platform.git
cd cubes-hr-platform

# 2. تثبيت المكتبات
pnpm install

# 3. إعداد قاعدة البيانات
# تأكد من تشغيل MySQL أولاً
mysql -u root -p < setup.sql

# 4. تشغيل التطبيق
pnpm dev

# 5. افتح المتصفح
# http://localhost:3000
```

### الطريقة 2: Docker (الأسهل)

```bash
# 1. استنساخ المشروع
git clone https://github.com/your-org/cubes-hr-platform.git
cd cubes-hr-platform

# 2. تشغيل Docker Compose
docker-compose up -d

# 3. افتح المتصفح
# http://localhost:3000
```

## 🎯 الخطوات الأولى

### 1. تسجيل الدخول

```
1. افتح http://localhost:3000
2. انقر على "تسجيل الدخول"
3. استخدم بيانات المدير أعلاه
4. انقر على "دخول"
```

### 2. إضافة موظف جديد

```
1. من القائمة الجانبية، انقر على "الموظفين"
2. انقر على زر "إضافة موظف جديد"
3. أدخل البيانات:
   - الاسم: أحمد محمد
   - البريد الإلكتروني: ahmed@example.com
   - الرقم الوظيفي: EMP001
   - المسمى الوظيفي: مهندس برمجيات
4. انقر على "حفظ"
```

### 3. تسجيل الحضور

```
1. من القائمة الجانبية، انقر على "الحضور"
2. اختر الموظف من القائمة
3. انقر على "حاضر" أو "غائب"
4. انقر على "حفظ"
```

### 4. إنشاء طلب موافقة

```
1. من القائمة الجانبية، انقر على "الموافقات"
2. انقر على "طلب موافقة جديد"
3. اختر نوع الطلب (إجازة، عقد، إلخ)
4. أدخل التفاصيل
5. انقر على "إرسال"
```

### 5. الموافقة على الطلبات

```
1. من القائمة الجانبية، انقر على "الموافقات"
2. اختر طلب من القائمة
3. انقر على "موافقة" أو "رفض"
4. أضف ملاحظات (اختياري)
5. انقر على "تأكيد"
```

## 🆘 المشاكل الشائعة

### المشكلة: "لا يمكن الدخول"

**الحل:**
- تأكد من تشغيل الخادم (`pnpm dev`)
- امسح ذاكرة التخزين المؤقت
- أعد تحميل الصفحة

### المشكلة: "خطأ في قاعدة البيانات"

**الحل:**
- تأكد من تشغيل MySQL
- تحقق من بيانات الاتصال في `.env.local`
- شغل `pnpm db:push`

### المشكلة: "الصفحة بطيئة"

**الحل:**
- تحقق من اتصال الإنترنت
- أغلق التطبيقات الأخرى
- أعد تشغيل الخادم
